# 파이썬 정규표현식 패턴 문법 총정리 — 시리즈 완결편

드디어 마지막 편이다. 1탄에서 `re.match()`, 2탄에서 4대장, 3탄에서 `re.compile()`과 플래그를 다루는 동안, 패턴 안의 `\d`, `\w`, `+`, `{3,4}` 같은 기호들은 그때그때 눈치껏 설명하고 넘어갔다. 오늘은 이 기호들을 처음부터 체계적으로 정리한다. 이 글을 읽고 나면 지금까지 시리즈에 나온 모든 패턴을 스스로 해석할 수 있게 된다.

패턴 문법은 크게 다섯 덩어리다. **① 문자 클래스 ② 수량자 ③ 위치 지정 ④ 그룹과 선택 ⑤ 이스케이프.** 순서대로 간다.

---

## 1. 문자 클래스 — "어떤 글자를" 잡을 것인가

문자 클래스는 "이 자리에 올 수 있는 글자의 종류"를 정하는 문법이다.

| 표현 | 의미 | 예시 매칭 |
|------|------|-----------|
| `\d` | 숫자 하나 (0-9) | `7` |
| `\w` | 단어 문자 하나 (영문자, 숫자, _) | `S`, `9`, `_` |
| `\s` | 공백 문자 하나 (스페이스, 탭, 줄바꿈) | ` ` |
| `.` | 아무 문자 하나 (줄바꿈 제외) | `가`, `@`, `3` |
| `[abc]` | a, b, c 중 하나 | `b` |
| `[a-z]` | a부터 z까지 중 하나 | `k` |
| `[^abc]` | a, b, c가 **아닌** 하나 | `d`, `1` |

`\d`, `\w`, `\s`는 각각 대문자 버전도 있는데, **대문자는 정확히 반대**를 뜻한다. `\D`는 숫자가 아닌 것, `\W`는 단어 문자가 아닌 것, `\S`는 공백이 아닌 것이다.

대괄호 `[ ]`는 "이 중 하나"를 직접 지정하는 문법이다. 시리즈 내내 썼던 패턴들이 이제 읽힐 것이다.

```python
import re

# 1탄의 아이디 패턴에 나왔던 [a-z0-9_]
# → 소문자 or 숫자 or 언더스코어 중 하나
print(re.findall(r"[a-z0-9_]", "SungJin_99"))
# ['u', 'n', 'g', 'i', 'n', '_', '9', '9'] (대문자 S, J는 제외됨)

# 2탄의 이메일 패턴에 나왔던 [\w.]
# → 단어 문자 or 마침표 중 하나
print(re.findall(r"[\w.]+", "sungjin.dev@gmail.com"))
# ['sungjin.dev', 'gmail.com'] (@에서 끊긴다)
```

대괄호 맨 앞의 `^`는 "부정"이다. `[^0-9]`는 숫자가 아닌 글자를 뜻한다. 뒤에서 볼 위치 지정의 `^`(줄 시작)와 기호는 같지만 **대괄호 안에 있을 때만** 부정의 의미라는 점에 주의하자.

## 2. 수량자 — "몇 개나" 잡을 것인가

문자 클래스가 종류를 정했다면, 수량자는 **반복 횟수**를 정한다. 항상 바로 앞의 표현 하나에만 적용된다.

| 표현 | 의미 |
|------|------|
| `+` | 1개 이상 |
| `*` | 0개 이상 |
| `?` | 0개 또는 1개 (있어도 되고 없어도 됨) |
| `{3}` | 정확히 3개 |
| `{3,4}` | 3개 이상 4개 이하 |
| `{3,}` | 3개 이상 |

시리즈 단골이었던 전화번호 패턴을 이제 완전히 해부할 수 있다.

```python
import re

# 01[016789]-\d{3,4}-\d{4}
#
# 01         → 글자 그대로 "01"
# [016789]   → 0,1,6,7,8,9 중 하나  (010, 011, 016...)
# -          → 글자 그대로 하이픈
# \d{3,4}    → 숫자 3~4개          (중간 번호)
# -          → 하이픈
# \d{4}      → 숫자 정확히 4개     (끝 번호)

print(re.match(r"01[016789]-\d{3,4}-\d{4}", "010-1234-5678"))  # 매치 성공
```

`?`는 "선택적 요소"를 표현할 때 진가가 나온다. 예를 들어 SungJin의 블로그 독자가 전화번호를 하이픈 없이 입력할 수도 있다면 이렇게 쓴다.

```python
import re

pattern = r"01[016789]-?\d{3,4}-?\d{4}"
#                    └ 하이픈이 있어도(1개) 없어도(0개) 매치

print(re.match(pattern, "010-1234-5678"))   # 성공
print(re.match(pattern, "01012345678"))     # 성공
```

### 함정 하나 — 탐욕적 매칭

수량자는 기본적으로 **최대한 길게** 잡으려 한다. 이걸 탐욕적(greedy) 매칭이라고 부른다.

```python
import re

html = "<b>SungJin</b>의 <b>블로그</b>"

print(re.findall(r"<b>.*</b>", html))
# ['<b>SungJin</b>의 <b>블로그</b>'] ← 통째로 하나! 첫 <b>부터 마지막 </b>까지 삼켰다

print(re.findall(r"<b>.*?</b>", html))
# ['<b>SungJin</b>', '<b>블로그</b>'] ← 수량자 뒤에 ?를 붙이면 최소로 잡는다
```

수량자 뒤의 `?`는 "게으른(lazy) 매칭으로 전환"이라는 뜻이다. 위의 "0개 또는 1개"를 뜻하는 `?`와 기호는 같지만 위치가 다르다. `.*?`처럼 **수량자 바로 뒤**에 붙으면 게으른 매칭이다. 태그나 따옴표처럼 짝이 있는 것을 잡을 때 필수로 등장한다.

## 3. 위치 지정 — "어디에서" 잡을 것인가

글자를 소비하지 않고 **위치만** 지정하는 기호들이다.

| 표현 | 의미 |
|------|------|
| `^` | 문자열(또는 줄)의 시작 |
| `$` | 문자열(또는 줄)의 끝 |
| `\b` | 단어 경계 |

`^`와 `$`는 3탄의 `re.MULTILINE`에서 이미 만났다. 새 얼굴은 `\b`인데, "단어와 단어 아닌 것의 경계"를 뜻한다. 부분 일치를 막고 싶을 때 쓴다.

```python
import re

text = "sungjin과 sungjin99와 mysungjin이 있다"

print(re.findall(r"sungjin", text))
# ['sungjin', 'sungjin', 'sungjin'] ← 셋 다 잡힘 (부분 일치)

print(re.findall(r"\bsungjin\b", text))
# ['sungjin'] ← 정확히 sungjin이라는 단어만
```

`sungjin99`는 뒤에 단어 문자(9)가 이어지므로 경계가 아니고, `mysungjin`은 앞에 단어 문자(y)가 붙어 있으므로 경계가 아니다. 금지어 필터에서 "grass에 ass가 들어 있다고 차단하는" 참사를 막아주는 게 바로 `\b`다.

한 가지 정리하고 갈 것. 1탄에서 `re.match()`는 맨 앞을 검사한다고 했는데, 그러면 `^`는 왜 필요할까? `match()`를 쓸 때는 `^`가 사실상 필요 없다. 하지만 `search()`나 `findall()`로 줄 시작을 잡을 때(특히 `re.M`과 함께), 그리고 끝을 고정하는 `$`와 짝을 맞출 때 필요하다. 형식 검증 패턴에 습관처럼 `$`를 붙였던 이유가 이것이다. `$`가 없으면 `"010-1234-5678은 가짜"` 같은 문자열도 앞부분만 맞아서 통과해 버린다.

## 4. 그룹과 선택 — 묶고, 고르기

### 그룹 ( ) — 이미 아는 것 + 한 가지 더

괄호가 "따로 꺼내 쓰는 그룹"이라는 건 1탄부터 써왔다. 그런데 괄호에는 용도가 하나 더 있다. **수량자를 여러 글자에 한꺼번에 적용**하는 것이다.

```python
import re

# ha+  → h 다음 a가 1개 이상   ("haaa")
# (ha)+ → "ha"라는 덩어리가 1개 이상 ("hahaha")

print(re.match(r"ha+", "haaaa").group())     # haaaa
print(re.match(r"(ha)+", "hahaha").group())  # hahaha
```

수량자는 "바로 앞의 표현 하나"에 붙는다고 했는데, 괄호로 묶으면 그 덩어리 전체가 "표현 하나"가 되는 원리다.

### 선택 | — "이것 아니면 저것"

`|`는 여러 후보 중 하나를 뜻한다. 3탄에서 플래그를 합칠 때 쓴 `|`와 기호는 같지만, 패턴 안에서는 "or"의 의미다.

```python
import re

text = "SungJin은 python과 Java와 rust를 공부한다"

print(re.findall(r"python|java|rust", text, re.IGNORECASE))
# ['python', 'Java', 'rust']
```

`|`는 범위가 넓게 잡히므로 보통 괄호와 함께 쓴다. `r"(png|jpg|gif)$"`처럼 쓰면 "png 또는 jpg 또는 gif로 끝나는"이라는 뜻이 된다.

```python
import re

IMAGE = re.compile(r"\w+\.(png|jpg|gif)$", re.IGNORECASE)

for name in ["profile.PNG", "sungjin_photo.jpg", "notes.txt"]:
    print(name, "→", "이미지" if IMAGE.match(name) else "이미지 아님")
# profile.PNG → 이미지
# sungjin_photo.jpg → 이미지
# notes.txt → 이미지 아님
```

## 5. 이스케이프 — 기호를 "글자 그대로" 쓰고 싶을 때

여기까지 오면 자연스러운 의문이 생긴다. `.`, `+`, `?`, `(`, `$` 같은 기호가 전부 특수 기능을 가진다면, **진짜 마침표나 진짜 물음표는 어떻게 잡을까?** 앞에 백슬래시를 붙이면 된다.

```python
import re

price = "가격: 3.50달러 (할인가)"

print(re.search(r"\d+.\d+", price).group())    # 3.50 → 되긴 하는데...
print(re.search(r"\d+\.\d+", price).group())   # 3.50 → 이게 정확하다
```

첫 번째 패턴도 결과는 같아 보이지만 함정이 있다. `.`은 아무 문자이므로 `"3x50"` 같은 잘못된 값도 통과시킨다. **소수점, 파일 확장자의 점, 도메인의 점은 반드시 `\.`으로** 써야 한다. 위 이미지 확장자 예제에서 `\w+\.`라고 쓴 이유다.

이스케이프가 필요한 주요 기호: `. + * ? ( ) [ ] { } ^ $ | \`

그리고 이제 시리즈 1탄부터 강조해온 `r"..."`의 정체도 완전히 설명된다. 패턴에는 백슬래시가 이렇게나 많이 등장하는데(`\d`, `\b`, `\.`), raw string이 아니면 파이썬 문자열 규칙과 충돌해서 `\\d`처럼 백슬래시를 두 번 써야 하는 지옥이 열린다. **`r`은 그 지옥을 막는 습관**이었던 것이다.

## 6. 최종 시험 — 시리즈의 모든 패턴 해석하기

시리즈에 나왔던 패턴 중 가장 복잡했던 것을 스스로 읽어보자. 1탄의 아이디 검증 패턴이다.

```
[a-z][a-z0-9_]{3,15}$
```

- `[a-z]` → 소문자 중 하나 (문자 클래스): 첫 글자는 영문 소문자
- `[a-z0-9_]` → 소문자·숫자·언더스코어 중 하나 (문자 클래스)
- `{3,15}` → 앞의 것이 3~15개 (수량자)
- `$` → 여기서 끝 (위치 지정)

읽힌다면 합격이다. 이제 처음 보는 패턴을 만나도 다섯 덩어리(클래스 → 수량자 → 위치 → 그룹/선택 → 이스케이프) 순서로 쪼개면 해석할 수 있다.

## 마무리 — 시리즈를 닫으며

네 편에 걸친 여정을 한 장으로 압축하면 이렇다.

| 편 | 배운 것 | 핵심 한 줄 |
|----|---------|-----------|
| 1탄 | `re.match()` | 맨 앞 검사, 결과는 Match 객체 or None |
| 2탄 | 4대장 비교 | 검증은 match, 탐색은 search, 수집은 findall, 치환은 sub |
| 3탄 | compile과 플래그 | 패턴은 미리 만들고, 규칙은 플래그로 바꾼다 |
| 4탄 | 패턴 문법 | 클래스·수량자·위치·그룹·이스케이프 다섯 덩어리 |

이 정도면 실무에서 마주치는 정규표현식의 대부분을 읽고 쓸 수 있다. 남은 건 손에 익히는 것뿐이다. 좋은 연습 방법 하나를 추천하자면, 자기가 매일 다루는 텍스트(로그, 주소록, 메모)에서 뭔가를 뽑아내는 작은 스크립트를 만들어 보는 것이다. 패턴은 눈으로 외우는 게 아니라 손으로 익히는 것이다.
