 # 자바 연산자 정리 노트
 
## 1. 연산의 기본 개념

**연산** : 프로그램에서 데이터를 처리하여 결과를 산출하는 것을 말한다.<br> 
**연산자 (operator)** : 연산에 사용되는 기호 (+, ==, && 등)<br>
**피연산자 (operand)** : 연산 대상이 되는 데이터<br>
<br>
**특징** <br>
연산식은 반드시 하나의 값을 산출한다
연산식은 다른 연산식의 피연산자 자리에 올 수 있다
<br>
java  boolean result = (x + y) < 5;   // (x+y)의 결과값이 <의 피연산자가 됨

## 2. 연산자 분류 

**피연산자 개수**: 단항(++, -, !) / 이항(+, ==) / 삼항(? :)<br>
**기능별**: 산술 → 비교 → 논리 → 조건(삼항) → 대입 (대체로 우선순위 순서이기도 함)<br>
**연산 방향**: 대부분 왼쪽 → 오른쪽. 단, 단항 연산자(++ -- + - ~ !)와 대입 연산자(=, +=)는 오른쪽→왼쪽

## 3. 타입 승격 (⭐ 핵심 규칙)

**int 미만 타입**(byte, short, char)의 연산 결과는 무조건 int — `정수 승격(integer promotion)`<br>
부호 연산자(-b)도 "연산"이므로 이 규칙 적용<br>
피연산자 중 long이 있으면 전체가 long으로, double이 있으면 double로 변환<br>
단, 리터럴끼리의 연산은 컴파일러가 미리 계산(상수 접기)하므로 범위만 맞으면 OK<br>
<br>
javabyte r1 = 5 + 7;    // 컴파일러가 미리 12로 계산, 범위 검사 통과<br>
byte a = 5, b = 7;<br>
byte r2 = a + b;    // 변수 연산 → int 승격 → 에러

## 4. 증감 연산자

전위 `++val`: "일단 더하고 시작!" (먼저 증가, 그 값을 사용)<br>
후위 `val++`: "지금은 그냥 쓰고, 나중에 더해!" (현재 값 사용 후 증가)

## 5. 나눗셈의 위험 지대 (⭐)
연산결과 5 / 0 (정수) -> 컴파일은 OK, 실행 시 `ArithmeticException` → 프로그램 즉시 종료<br>
5 % 0 (정수)위와 동일<br>
5 / 0.0 (실수) -> Infinity (예외 없음!)<br>
0.0 / 0.0 등 -> NaN (Not a Number)<br>
<br>
예외 대비: try/catch 블록, 또는 연산 전 값 검사 (`safeAdd()`처럼 오버플로우 예상 시 예외 발생시키는 패턴)<br>
Infinity/NaN이 나오면 후속 연산 금지 → if로 막기<br>
검사 메서드: Double.isInfinite(x), Double.isNaN(x) → boolean 반환<br>
NaN은 ==로 검사 불가 — NaN은 자기 자신과도 같지 않아서, !=를 제외한 모든 비교 연산이 false를 반환

## 6. 비교 연산의 함정들 (⭐)
java3 == 3.0        // true  (int가 double로 승격되어 3.0 == 3.0)<br>
0.1 == 0.1f     // false! (float와 double 모두 0.1을 정확히 표현 못 하는데, 근사값이 서로 다름)<br>
<br>

String 비교:

<, > 대소 비교 불가 (컴파일 에러)<br>
== 는 참조(주소) 비교 — 문자열 리터럴이 같으면 같은 String 객체를 참조하므로 true가 나오기도 하지만, 믿으면 안 됨<br>
내용 비교는 반드시 `equals()`: boolean result = str1.equals(str2);

## 7. 문자열 연결 + (왼쪽 → 오른쪽 순서가 핵심)
java"jdk" + 3 + 3.0   // "jdk3" → "jdk33.0"  (문자열 먼저 만나면 이후 전부 연결)<br>
3 + 3.0 + "jdk"   // 6.0 → "6.0jdk"      (숫자끼리 먼저 계산됨)

## 8. 논리 연산자 (피연산자는 boolean만)
연산자이름규칙&& / &AND (논리곱)모두 true여야 true|| / |OR (논리합)하나라도 true면 true^XOR (배타적 논리합)둘이 다르면 true, 같으면 false!NOT (논리부정)반전
<br>
&&는 앞이 false면 뒤를 아예 검사 안 함 (단축 평가) → 효율적 + 안전
&는 둘 다 무조건 검사 → 조건문에서는 &&, || 사용이 원칙

## 9. 비트 연산자 (0과 1로 표현 가능한 정수 타입만)
연산자규칙& AND모두 1일 때만 1| OR하나만 1이어도 1^ XOR값이 다르면 1~ NOT반전<< >>비트 이동 (1 << 3 = 8, 왼쪽 한 칸 = ×2)
<br>
편의 메서드: `Integer.toBinaryString(n)` → 10진수를 2진수 문자열로 변환

## 10. 대입 연산자

단순 대입 =, 복합 대입 +=, -=, *= 등<br>
모든 연산자 중 우선순위가 가장 낮음 (모든 계산이 끝난 후 마지막에 대입)<br>
연산 방향: 오른쪽 → 왼쪽
<br><br>
참고 ) 
javaSystem.out.println(7 / 0.0);   // Infinity 출력, 예외 없음!
나누는 수가 정수 0이 아니라 **실수 0.0**. 부동소수점 나눗셈은 0으로 나눠도 예외가 발생하지 않고 Infinity라는 특수값을 내놓음. (참고로 0.0 / 0.0이면 NaN)
정리하면:
정수 / 0실수 / 0.0결과 ArithmeticException (프로그램 종료) Infinity 또는 NaN (조용히 계속 진행)
실수 쪽이 오히려 더 위험 — 예외로 멈추지 않고 Infinity가 후속 계산에 스며듬. 그래서 "Infinity/NaN 검사 후 후속 연산 차단"이 필요
