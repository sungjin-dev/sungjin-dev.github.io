C언어에서 가장 당황스러운 순간 중 하나가 바로 이 name = "SungJin";이 안 된다는거다. 

그 이유를 메모리 관점에서 살펴보자. 

## 1. 왜 name = "SungJin"은 안 될까?

C언어에서 배열(문자열)은 선언될 때 '딱 정해진 메모리 주소'를 할당받는다.

C
char name[20]; // 메모리에 "name"이라는 이름의 20바이트 공간이 확보됨

name = "SungJin";  // 에러 발생

여기서 name은 20바이트 공간의 **'시작 주소'**를 가리키는 `상수`이기 때문에, 그 주소를 가리키는 고정된 이름을 다른 주소를 가리키지 못하게 막는 것이다. 

 이런 경에 String Copy(문자열 복사)를 이용하면 된다. 

## 2. strcpy 사용법

`strcpy`는 주소를 바꾸는 게 아니라, 해당 메모리 주소로 찾아가서 안에 있는 데이터를 덮어쓰라는 명령이다.

strcpy(name, "SungJin") : "현재 주소로 (SungJin)을 들고 가서, 너를 채워 넣어라" (성공!)


## 3. C언어의 낭만(?) - 깐깐함의 이유

파이썬이나 자바스크립트는 name = "SungJin"이라고 하면, 내부적으로 메모리 주소를 갈아치우는 작업을 엔진이 알아서 해준다. 

하지만 C언어는 "메모리 주소는 프로그래머가 관리해야 한다"는 철학 때문에, 프로그래머가 직접 strcpy라는 복사 명령을 내려야만 비로소 메모리 내용이 바뀐다. 이렇게 함으로써 데이터가 어디서 어디로 복사되는지를 프로그래머가 완전히 통제할 수 있게 되는 것이죠.


"문자열 변수에 값을 직접 대입(=)할 수 없기 때문에"

정리: strcpy는 단순히 복사하는 게 아니라, "직접 주소로 찾아가서 덮어쓰는 것"이다!


가비지 컬렉터는 "알아서 구석구석 먼지(쓰레기 메모리)를 치워주는 로봇청소기"이고, C언어 프로그래머는 "내 집 구조와 먼지가 어디에 쌓이는지 완벽히 파악하고, 직접 빗자루질을 하는 장인"이라고 할 수 있겠네요.
