## 1. 학습 준비물 

Roboflow에서 라벨링을 마치고 데이터셋을 다운로드(Export)할 때 포맷을 선택할 수 있는데, 그때 'COCO' 포맷을 선택하면 JSON 파일(_annotations.coco.json 등)을 주는데, 

이 COCO JSON 파일을 YOLO용 txt 파일로 변환할 때 쓰는 전형적인 코드다. 

```python

import json
import os
import sys

BASE = "C:/Alot1team/project/yolo_trainning/dataset"

SPLITS = {
    "train": {
        "json_path": f"{BASE}/raw/instances_train.json",
        "output_dir": f"{BASE}/labels/train",
    },
    "val": {
        "json_path": f"{BASE}/raw/instances_val.json",
        "output_dir": f"{BASE}/labels/val",
    },
}

  # categories 정의 기준으로 YOLO class index 매핑 생성

def build_id_map(categories):
    id_map = {}
    new_idx = 0
    for cat in sorted(categories, key=lambda c: c["id"]):
        id_map[cat["id"]] = new_idx
        new_idx += 1
    return id_map

def convert(split_name, json_path, output_dir, expected_id_map=None):
    print(f"\n{'=' * 50}")
    print(f"[{split_name}] {json_path}")
    print(f"{'=' * 50}")

    with open(json_path, "r", encoding="utf-8") as f:
        data = json.load(f)

    # 1) 클래스 매핑 (categories 정의 기준)
    id_map = build_id_map(data["categories"])
    name_by_id = {c["id"]: c["name"] for c in data["categories"]}

    print("클래스 매핑 (원본 category_id -> YOLO class index):")
    for old_id, new_id in id_map.items():
        print(f"  {old_id} ({name_by_id[old_id]}) -> {new_id}")

    # 스플릿 간 매핑 일치 검증 (all 모드에서 자동 수행)
    if expected_id_map is not None and id_map != expected_id_map:
        raise ValueError(
            f"[{split_name}] 클래스 매핑이 train과 다릅니다! "
            f"json의 categories 정의가 스플릿마다 다른지 확인하세요.\n"
            f"  train: {expected_id_map}\n  {split_name}: {id_map}"
        )

    # 2) image_id -> 이미지 정보 매핑
    images = {img["id"]: img for img in data["images"]}

    # 3) image_id 별로 annotation 모으기
    anns_by_image = {img_id: [] for img_id in images}
    skipped_unknown = 0
    
    for ann in data["annotations"]:
        cid = ann["category_id"]
        if cid not in id_map:
            # categories에 정의되지 않은 id가 annotation에 등장한 경우 (비정상 데이터)
            skipped_unknown += 1
            continue
        anns_by_image[ann["image_id"]].append(ann)

    if skipped_unknown:
        print(f"[경고] categories에 정의되지 않은 category_id를 가진 "
              f"annotation {skipped_unknown}개를 건너뜀 — 데이터 확인 필요!")

    # 4) 출력 폴더 생성
    os.makedirs(output_dir, exist_ok=True)

    # 5) 이미지 전체에 대해 txt 생성 (객체 없는 이미지는 빈 txt = 배경 이미지)
    written = 0
    for img_id, img in images.items():
        w, h = img["width"], img["height"]
        lines = []
        for ann in anns_by_image[img_id]:
            x, y, bw, bh = ann["bbox"]  # COCO: top-left x, y, width, height
            cx = (x + bw / 2) / w
            cy = (y + bh / 2) / h
            nw = bw / w
            nh = bh / h
            cls = id_map[ann["category_id"]]
            lines.append(f"{cls} {cx:.6f} {cy:.6f} {nw:.6f} {nh:.6f}")

        base_name = os.path.splitext(img["file_name"])[0]
        out_path = os.path.join(output_dir, base_name + ".txt")
        with open(out_path, "w", encoding="utf-8") as f:
            f.write("\n".join(lines))
        written += 1

    print(f"총 {written}개의 txt 라벨 파일 생성 완료 -> {output_dir}")
    return id_map

def main():
    if len(sys.argv) != 2 or sys.argv[1] not in ("train", "val", "all"):
        print("사용법: python coco_to_yolo.py [train | val | all]")
        sys.exit(1)

    target = sys.argv[1]

    if target == "all":
        # train을 기준 매핑으로 삼고, val이 일치하는지 자동 검증
        train_map = convert("train", **SPLITS["train"])
        convert("val", **SPLITS["val"], expected_id_map=train_map)
        print("\n[OK] train/val 클래스 매핑 일치 확인 완료")
    else:
        convert(target, **SPLITS[target])

if __name__ == "__main__":
    main()
```

YOLO 모델을 학습시키려면 이미지마다 개별적인 .txt 라벨 파일이 필요한데, 이 코드는 그 업무를 포함하여 4가지 핵심적인 역할을 수행한다.

**1. 클래스 인덱스 0부터 재배열 (Class Mapping)**

COCO 데이터셋은 클래스 ID가 1부터 시작하거나 중간에 빠진 번호가 있을 수 있다(예: 1, 2, 4, 7). 하지만 YOLO는 무조건 0부터 시작하는 연속된 정수(0, 1, 2, 3)를 요하는데, 

이 때 JSON 파일의 categories를 읽어와서 기존 ID를 YOLO가 인식할 수 있도록 0, 1, 2... 순서로 새롭게 매핑(id_map)해준다. 

**2. 바운딩 박스 좌표 변환 (정규화)**

두 포맷은 박스를 그리는 방식이 완전히 다르다.

COCO: [좌상단 x, 좌상단 y, 너비, 높이] (실제 픽셀 값)

YOLO: [중심 x, 중심 y, 너비, 높이] (이미지 전체 크기 대비 0~1 사이의 비율 값)

코드 내의 수식(cx = (x + bw / 2) / w 등)을 통해 COCO의 절대 좌표를 YOLO 방식의 상대적(정규화된) 중심 좌표로 계산하여 변환한다.

**3. 개별 TXT 라벨 파일 생성**

JSON 파일 하나에 뭉쳐 있던 수천 장의 이미지 정보를 분리하여, 이미지 파일명과 똑같은 이름의 .txt 파일을 labels/train 또는 labels/val 폴더에 생성한다.

배경 이미지 처리: 라벨링된 객체가 없는 이미지라도 빈 .txt 파일을 만들어낸다. (YOLO는 이를 '아무것도 없는 배경 이미지'로 정상 인식하여 오탐지를 줄이는 데 활용한다.)

**4. Train / Val 클래스 불일치 검증**

python coco_to_yolo.py 명령어로 실행할 경우, 훈련용(train) 데이터와 검증용(val) 데이터의 클래스 순서가 똑같이 매핑되었는지 자동으로 비교한다.

만약 두 JSON의 클래스 구성이 달라서 인덱스가 꼬이게 되면 에러를 띄워 학습 망침을 사전에 방지하는 것이다. 

즉, 별다른 후처리 없이 바로 학습만 돌릴거라면 Roboflow에서 애초에 YOLO 형식으로 Export 하시는 것이 가장 빠르고 에러도 안 난다. 


## 2. 실행 코드

다음은 실행할 main.py다. 주로 train.py로 쓴다. 

```python

from ultralytics import YOLO

def start_training():
   
    model = YOLO('yolov8n.pt') 
    # model = YOLO(r'C:\Alot1team\project\yolo_trainning\results\my_experiment_v1-5\weights\last.pt')
  
    model.train(
        data=r'C:\Alot1team\project\yolo_trainning\data.yaml', 
        epochs=120,          # 전체 데이터셋을 반복 학습할 총 횟수
        imgsz=640,           # 모델에 입력되는 이미지 크기
        patience=90,         # Early Stopping(조기 종료)
        augment=True,        # 증강 활성화
        mosaic=0,            # 4분할 모자이크 
        degrees=5,           # 회전 적용
        perspective=0.001,   # 원근 변환 적용
        shear=0,             # 기울기 적용
        fliplr=0,            # 좌우 대칭
        batch=16,            # 1회 연산 시 처리할 이미지 수
        project=r'C:\Alot1team\project\yolo_trainning\results',  
        name='my_experiment_v1'                 
    )

if __name__ == '__main__':

    # model = YOLO(r'C:\Alot1team\project\yolo_trainning\results\my_experiment_v1-5\weights\last.pt')
    # model.train(resume=True)

    start_training()

```

주석으로 달긴 했지만 각 파라미터들을 다시 한번 정리해보자.  

`epochs` : 전체 데이터를 총 몇 번 반복해서 학습할지 정하는 횟수

`imgsz`: 모델이 학습할 때 사용할 이미지의 해상도 

`patience` : ~번의 에포크 동안 성능 개선이 없으면 학습을 강제 종료 (과적합 방지)

`augment(True)` : 데이터 변형(증강) 옵션을 켜서 학습 데이터를 다양하게 만듦

`mosaic` : 4장의 이미지를 하나로 바둑판처럼 합치는 기능 

`degrees` : 이미지를 왼쪽/오른쪽으로 최대 해당 수치만큼 랜덤하게 회전시킴

`perspective` : 이미지에 아주 미세한 3D 입체(원근감) 왜곡을 줌

`shear` : 이미지를 평행사변형처럼 비틀고 기울이는 기능 

`fliplr` : 이미지를 거울처럼 좌우로 뒤집는 기능 

`batch` : 그래픽카드가 한 번의 연산에 동시에 처리할 이미지의 개수 


주로 epochs, imgsz, patience, batch만 만져도 충분하긴 하다. 토이프로젝트를 할 때 RTX 2060으로 돌렸었는데, 해상도를 1280으로 하니까 본체가 너무 뜨거워져서 방이 후끈했던 기억이 난다. 

imgsz를 올릴 때는 배치 사이즈를 1/2로 낮춰서 진행했다. 그런데 어차피 ESP32를 사용해서 640으로 실시간 스트리밍 영상 프레임으로 객체탐지를 진행하기 때문에 다음 학습 때는 그냥 640으로 진행했다. 

imgsz가 클수록 추론 연산량(FLOPs)이 커져서 FPS가 떨어지기 때문인데, 실시간 스트리밍 처리에서는 특히나 치명적이다. 굳이 1280으로 처리하면 필요 없는 지연만 늘어나는 역효과가 날 뿐이다. 


그리고 설명을 제외한 주석으로 남긴 부분들은 중간에 예기치 않게 학습이 중단된 경우 resume=True와 함께 `last.pt`을 이용하여 재개할 수 있다. 

best.pt가 가장 최고의 학습데이터를 가지고 있다면 last.pt는 가장 최근까지의 모든 학습데이터를 담고 있다.
